Known bugs and issues

  • MiMI Internal ID
    MiMI internal IDs are build dependent. This affects users node/edge annotations since we use this internal id as the identifier of node/edge annotation. It can also affect external storage and linking of data using MiMI IDs.

  • MiMI merging issues
    For some molecules, MiMI assigns them to different internal IDs although they have same molecule names. For example, when users query molecule “csf1r” from MiMI for mouse, protein with interaction only from BIND data source, they will get two nodes called “csf1r” but with 46646 and 5983 as the molecule IDs.

For bug reporting please email: mimi-help@umich.edu
